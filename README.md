## 📌 Project Overview

This project is an audit and analysis of **Git**, a distributed version control system.  
The aim of this project is to understand the philosophy, licensing, and real-world usage of open-source software.

Along with theoretical analysis, this project also includes practical implementation using Linux and shell scripting.

---

## 💡 Why I Chose Git

While working on coding projects, I realized that managing different versions of code manually was difficult.  
That’s when I started learning Git, and it helped me track changes and manage my work efficiently.

So, I decided to explore Git in depth for this project.

##  Shell Scripts

This project includes 5 shell scripts:

### 🔹 Script 1: System Identity Report
Displays system details like kernel version, user, uptime, and distribution.

### 🔹 Script 2: FOSS Package Inspector
Checks whether Git is installed and shows its version.

### 🔹 Script 3: Disk and Permission Auditor
Analyzes system directories and displays permissions, owner, and size.

### 🔹 Script 4: Log File Analyzer
Reads a log file and counts occurrences of a keyword.

### 🔹 Script 5: Open Source Manifesto Generator
Takes user input and generates a personalized text file.
